# 智能知识库平台 - 项目概述

## 第一章 项目背景

智能知识库平台是一个面向团队内部文档集中管理、检索、问答和引用核验的综合平台。该平台旨在解决企业文档管理中的以下核心问题：文档分散在多种格式中无法统一检索、扫描件和图片缺乏可搜索文本、大模型回答缺少可核验引用、用户权限管理分散容易导致越权访问。

### 1.1 技术架构

平台采用前后端分离架构，前端使用 Vue 3 和 TypeScript 构建统一的用户界面，后端基于 FastAPI 提供高性能的 API 服务。数据存储采用 PostgreSQL 数据库配合 pgvector 扩展实现向量检索，Redis 用于任务队列和缓存。

### 1.2 核心功能

平台首期支持以下核心功能：用户认证与权限管理、多格式文档上传与自动转换、关键词检索、向量检索和混合检索、基于大模型的流式问答与引用溯源、文档五格式导出（PDF、DOCX、Markdown、CSV、TXT）以及完整的管理中心。

## 第二章 文档处理流程

文档上传后进入统一的处理流水线：上传校验 -> 格式识别 -> 文档转换 -> OCR（必要时）-> Markdown 标准化 -> 文本分段 -> 全文索引与向量索引 -> 就绪状态。每个阶段都有明确的状态标记和错误处理机制。

### 2.1 支持的格式

平台首期支持以下文档格式：PDF、Word（.docx/.doc）、Excel（.xlsx/.xls）、PowerPoint（.pptx/.ppt）、Markdown、纯文本、CSV、HTML、EPUB、开放文档格式（ODT/ODS/ODP）、图片（PNG/JPG/WebP/BMP/TIFF）、邮件（EML）等。

### 2.2 安全要求

所有文档处理过程中必须进行安全清洗：Markdown 内容需要服务端清洗和前端 DOMPurify 二次过滤，禁止脚本、事件属性和危险协议。上传文件需要校验扩展名、MIME 类型和文件头，防止路径穿越和恶意文件上传。

## 第三章 部署与运维

平台使用 Docker Compose 进行容器化部署，包含五个核心服务：Web（Nginx + Vue SPA）、API Server（FastAPI）、Worker（异步任务处理）、PostgreSQL（数据存储）和 Redis（缓存队列）。所有镜像使用精确版本标签，测试和生产环境使用相同的镜像。
