## 第一章 产品概述

### 1.1 产品定位

智能知识库平台是一个面向团队内部文档集中管理、检索、问答和引用核验的综合平台。平台旨在解决企业文档管理中的碎片化问题，提供从文档上传到智能问答的全链路解决方案。

### 1.2 目标用户

平台面向需要管理大量内部文档的团队，包括但不限于：

- 技术团队：管理技术文档、API 文档、开发规范
- 运营团队：管理运营手册、培训材料、客户案例
- 管理层：查看系统概览、审计日志、团队使用情况

### 1.3 核心价值

1. 统一文档管理：将分散在多种格式中的文档统一转换为可检索的标准化 Markdown
2. 智能检索：支持关键词、向量和混合检索，快速定位所需信息
3. 可信问答：基于检索增强生成（RAG）的流式问答，每条回答附带可核验引用
4. 安全管理：完善的权限体系和数据安全机制

## 第二章 技术架构

### 2.1 整体架构

平台采用前后端分离架构，由一个统一的 Vue 3 前端应用和一个 FastAPI 模块化后端组成。

```text
                         统一 Web 应用
              普通用户工作区 + 管理中心 /admin
                              |
                            Nginx
                 静态资源、SPA fallback、/api 代理
                              |
                       FastAPI API Server
                              |
       +----------------------+----------------------+
       |                      |                      |
   认证权限模块           知识文档模块           RAG 与导出模块
       |                      |                      |
       +----------------------+----------------------+
                              |
                    PostgreSQL + pgvector
                              |
                       本地文件存储卷
                              |
                    Redis 异步任务队列
                              |
                            Worker
```

### 2.2 部署单元

| 容器 | 职责 | 端口 | 健康检查 |
|---|---|---|---|
| web | Nginx + Vue SPA | 80 | HTTP 200 |
| api-server | FastAPI 接口 | 8000 | /api/v1/health/live |
| worker | 异步任务处理 | - | 进程存活 |
| postgres | 数据存储 | 5432 | pg_isready |
| redis | 缓存队列 | 6379 | redis-cli ping |

### 2.3 技术选型

| 层级 | 技术栈 | 固定版本 |
|---|---|---|
| 前端框架 | Vue 3 + TypeScript | 3.5.39 / 5.9.3 |

## 第三章 文档处理

### 3.1 处理流程

```text
uploaded -> detecting -> converting -> ocr -> normalizing -> chunking -> indexing -> ready
```

### 3.2 支持格式

平台首期支持以下文档格式的自动处理：

| 类型 | 扩展名 | 处理方式 |
|---|---|---|
| PDF | .pdf | 原生文本提取；扫描件 OCR |
| Word | .docx .doc | DOCX 解析；DOC 经 LibreOffice 转换 |
| Excel | .xlsx .xls | 工作表转表格 |
| PowerPoint | .pptx .ppt | 每页标题、正文和备注 |
| Markdown | .md | 清洗标题、链接、代码和图片 |
| 文本 | .txt .log | 编码识别并生成段落 |
| CSV | .csv | Markdown 表格 |
| HTML | .html | 提取正文和层级结构 |
| 图片 | .jpg .png .webp | OCR 和版面分析 |

## 第四章 检索与问答

### 4.1 检索模式

系统支持三种检索模式：

1. **关键词检索**：基于 PostgreSQL 全文检索，适用于精确名词和原文匹配
2. **向量检索**：基于 pgvector 相似度计算，适用于语义近似表达
3. **混合检索**：两路召回、归一化和融合，兼顾精确和语义

### 4.2 流式问答

问答接口使用 SSE 协议进行流式返回，事件序列如下：

```text
start -> delta* -> citation* -> done
```

### 4.3 引用溯源

每条回答附带引用信息，包含：

- 文档名称和 ID
- 页码和段落
- 原文片段和相关度分数
- 文档权限验证状态

## 第五章 安全与权限

### 5.1 认证机制

- 密码使用 Argon2 哈希
- Token 使用 HttpOnly Cookie
- 支持会话管理和 Token 刷新

### 5.2 权限模型

采用 RBAC（基于角色的访问控制）模型：

- 用户：拥有一个或多个角色
- 角色：包含一组权限码
- 权限码：控制页面访问和操作许可
- 知识库数据权限：控制文档级别的访问

### 5.3 安全措施

- Markdown 服务端清洗 + 前端 DOMPurify 二次过滤
- 文件上传校验扩展名、MIME 和文件头
- 模型密钥只写不回显
- 审计日志覆盖所有管理操作
- 导出文件鉴权下载，过期自动清理

## 附录 B：常见问题

**Q: 上传文件大小限制是多少？**
A: 默认 100MB，可通过环境变量配置。

**Q: 支持哪些语言的 OCR？**
A: 首期支持中文简体和英文，可通过安装额外的 Tesseract 语言包扩展。

**Q: 如何备份数据？**
A: 使用 scripts/backup.sh 进行数据库和文件存储备份，scripts/restore.sh 进行恢复。
