# 个人知识库、文档切分与命中率修正记录

## 交付结论

- 用户工作区的企业知识库不再提供上传入口；企业文档由管理中心上传。
- 每个既有和新建用户自动拥有一个仅本人可见的个人知识库，普通用户可在其中上传、检索和问答。
- 文档按“解析/OCR -> Markdown -> 清洗 -> 切分 -> embedding -> pgvector”处理，切分策略改为上传时选择。
- 命中率全零并非阈值问题，而是文档重处理后测试集仍引用旧分块 ID；本次已修复并恢复真实指标。

## 接口与数据修正

### 个人知识库

- `knowledge_bases` 新增 `kind` 与 `owner_user_id`。
- 企业库继续按“部门 + 规范化名称”唯一；个人库按所有者唯一。
- 迁移为每个既有用户补建个人库；用户注册和管理员新建用户时在同一事务中创建个人库。
- 超级管理员可管理全部企业库，但不能读取其他用户的个人库。
- 新增 `personal.document.upload` 权限，所有用户均可向自己的个人库上传。

### 文档处理

- `documents` 新增 `chunk_strategy`、`chunk_size`、`chunk_overlap`，参数随文档保存，便于重处理复现。
- 旧文档兼容标记为 `format`，不自动重建索引；新上传默认 `recursive`。
- `fixed` 使用固定字符窗口；`semantic` 按 Markdown 语义单元聚合；`recursive` 依次按段落和句子分隔符拆分；`format` 保留 Markdown 标题、段落、表格和代码块边界。
- 管理中心负责企业库上传；用户知识库详情只在个人库显示拖拽上传区和切分参数。

### 命中率测试

- 运行前检查相关分块是否仍为活动索引。
- 旧分块仅在“同一文档且内容完全相同”时映射到当前活动分块，并把新 ID 写回测试集。
- 无法精确映射时停止测试并提示重新标注，避免用检索结果反推标准答案造成评测泄漏。
- 阈值改为在重排后作用于最终相关性分数，不再用约 `0.016` 的 RRF 分数与 `0-1` 阈值直接比较。

## RAG 准确性与速度

准确性优先的当前链路为：混合召回（关键词 + 向量）-> RRF -> 低信息片段过滤 -> rerank -> 阈值过滤 -> LLM 基于引用生成答案。重排输入已加入文档标题和章节标题。

后续优先级：先维护高质量测试集并按文档版本重测；医疗结构化文档优先比较 `format` 与 `recursive`；保持混合召回和 rerank；用 embedding 批处理、pgvector HNSW 与答案缓存提速。只有真实问题集证明有效时，再增加查询改写等额外 LLM 调用。

## 验证记录

- Alembic `0018_personal_kb_chunk_strategy` 成功。
- 3 个既有用户生成 3 个个人库，缺失数为 0；14 份旧文档均保留 `format`。
- 普通用户只看到本部门企业库和本人个人库；超级管理员也只看到本人个人库。
- 医疗测试集由 `4/4` 失效标签修复为 `4/4` 活动标签；`hybrid / TopK 10 / rerank / threshold 0` 得到 Hit Rate `0.50`、MRR `0.15`。
- 后端定向测试：28 passed。前端 TypeScript、ESLint、生产构建通过，Vitest `88 passed`。
- 运行地址：[http://127.0.0.1](http://127.0.0.1)。

## 已知限制

- `semantic` 当前是确定性的段落/句子语义边界聚合，不额外调用大模型，便于复现并控制上传成本。
- 切换切分策略需要重处理文档；内容发生变化时测试标签仍需人工复标。
