# 发布检查清单

> 测试镜像继承生产运行时镜像，并仅追加锁定的开发依赖；发布前还需核对生产镜像摘要。
> 用途：每次发布前逐项确认，所有项通过后方可发布

---

## A. 代码和测试

- [ ] 所有 P0 Issue 已关闭，PR 已合并到 `main`
- [ ] CI `quality` job 最后一次运行全部通过
- [ ] 后端全量测试覆盖率不低于仓库门禁 `50%`，关键权限与安全边界有聚焦回归测试
- [ ] 所有契约测试通过，OpenAPI 与前端类型一致
- [ ] 所有集成测试通过（上传到检索、问答到引用、导出到下载）
- [ ] 所有安全测试通过（越权、上传、XSS、Secret 泄露、下载安全）
- [ ] 权限测试矩阵全部场景通过，管理接口权限校验 100%
- [ ] 前端 1440px、1280px 和 375px 视觉验证完成
- [ ] 没有未解决的严重或高危缺陷

## B. 镜像和环境

- [ ] 所有直接依赖使用精确版本（检查 `pyproject.toml`、`package.json`、Dockerfile）
- [ ] Web、API Server、Worker 镜像构建成功
- [ ] 镜像标签包含版本号和提交 SHA（如 `v1.0.0-abc1234`）
- [ ] 测试环境的运行时基础层与待发布镜像一致，并记录待发布镜像 SHA256
- [ ] 生产环境变量已准备（不包含在 Git 中）
- [ ] `deploy/env/.env.example` 与生产变量清单一致，测试专用非密钥变量与 `deploy/docker-compose.test.yml` 一致

## C. 数据库和备份

- [ ] 数据库迁移正向执行成功，回滚方案已验证
- [ ] 迁移在空数据库和已有数据数据库上都测试通过
- [ ] 最近一次备份成功完成并验证完整性
- [ ] 备份恢复在测试环境演练成功
- [ ] 备份文件不包含 Secret 明文

## D. 监控和健康检查

- [ ] 所有 5 个服务健康检查通过
- [ ] Prometheus 指标正常采集
- [ ] 告警规则已配置并测试（无大量误报）
- [ ] 日志正常输出，格式正确，不包含 Secret

## E. 文档和回滚

- [ ] `docs/operations/deployment.md` 是最新版本，可由非作者执行
- [ ] `docs/operations/backup-restore.md` 是最新版本，恢复演练已通过
- [ ] 回滚镜像已准备（上一个稳定版本的镜像）
- [ ] 回滚方案已确认：应用镜像回滚步骤、数据库回滚策略
- [ ] 发布失败应急联系人已确认

## F. 安全

- [ ] 严重安全问题为 0（越权、Secret 泄露、任意文件访问、存储型 XSS）
- [ ] 生产环境 TLS 证书已配置，Secure Cookie 标志已开启
- [ ] Nginx 安全响应头已配置
- [ ] 模型密钥不回显、不进入日志、不持久化到前端
- [ ] 上传目录禁止执行，文件下载不通过公开静态目录暴露
