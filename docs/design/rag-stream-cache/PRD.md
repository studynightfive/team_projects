# RAG 流式回答、处理摘要与相似问题缓存 PRD

## 背景

用户端 AI 搜索目前等待完整 RAG 请求结束后才展示答案。后端虽然已有会话流式接口，
但该接口会创建历史会话，与当前已取消历史会话页面的产品边界不一致。现有问答缓存只识别
完全相同的问题，且未绑定文档版本、模型和检索参数，知识库更新后存在复用旧答案的风险。

## 目标

1. AI 搜索通过独立的检索流式接口逐步展示答案，并支持用户取消。
2. 展示缓存检查、知识检索、结果重排、答案生成等可审计处理阶段。
3. 不展示或保存模型隐藏思维链、`reasoning_content` 或 `<think>` 内容。
4. 完全相同和高置信度近重复问题可以复用答案，并明确标识缓存类型。
5. 缓存必须受用户权限范围、知识库文档版本、模型和检索参数隔离。

## 用户流程

1. 用户选择一个有权访问的知识库并提交问题。
2. 页面立即进入处理状态，依次更新处理阶段。
3. 检索完成后先展示引用来源，随后逐段显示模型答案。
4. 用户可在生成期间取消；已展示的临时内容不写入问答缓存。
5. 缓存命中时直接展示已验证答案和来源，并说明是完全匹配或相似问题复用。

## 验收标准

- 流式接口使用 `text/event-stream`，首个 `start` 事件不等待模型完整生成。
- 事件覆盖 `start`、`stage`、`citation`、`delta`、`done` 和 `error`。
- 页面在首个 `delta` 到达后即可阅读答案，生成期间可取消。
- 网络失败、模型失败和用户取消都有明确收尾状态，不把部分答案标记为完成。
- 处理阶段只描述系统事实，不包含模型隐藏思维链。
- 相似缓存仅复用高置信度近重复问题；低于阈值时继续真实检索与生成。
- 缓存键包含用户、可访问知识库范围、文档版本、聊天模型、Embedding 模型、
  Rerank 模型及检索参数。
- 文档新增、替换、重新处理或删除后，旧缓存不再命中。
- Redis 不可用时问答降级为不使用缓存，不能导致 RAG 整体失败。
- 一次性 `/retrieval/answer` 保持兼容，并与流式接口使用同一缓存规则。

## 约束与风险

- 相似识别采用本地文本特征和高阈值，不额外调用 Embedding 模型，避免增加首字节延迟。
- 本期识别近重复表达，不承诺跨语言或完全不同措辞的语义等价。
- 缓存仍按用户隔离，不在不同用户之间共享答案。
- SSE 继续使用原生 Fetch 和 `ReadableStream`，不新增前端 SSE 依赖。
- 已有 `/chat/stream` 保留给会话能力，AI 搜索不依赖该接口。
