# AI 搜索工作台升级 PRD

## 1. 目标

在已批准的 M01 V2 用户工作区壳层内，把普通用户首页与知识检索页升级为面向企业内部员工的 AI 搜索工作台。用户应能从一个聚焦式自然语言入口完成提问，并在同一条可解释链路中查看 AI 答案、引用依据、原始结果和文档正文。

本升级仅取代 M01 的用户首页内容基线；M01 V2 的工作区壳层、设计 tokens、导航规则和基础交互规范继续作为事实来源，不另建主题或壳层。

本轮仍是确定性本地交互基线，不代表后端联调完成。所有搜索、研究、数据源和收藏状态，以及尚未由 OpenAPI 确认的字段、权限和连接状态，都必须明确标注为本地模拟；真实业务请求保持为 0。

本轮成果仅保留在本地工作区，未经用户观察、调整和明确同意，不推送到 GitHub，也不创建 Pull Request。

## 2. 用户价值

- 普通员工：用自然语言快速查找制度、项目、研发、财务和客户资料，不必先理解知识库目录。
- 研究型用户：查看检索范围、研究步骤和交叉验证来源，判断结论是否可信。
- 内容负责人：通过更新时间、负责人、权限和验证状态定位需要维护的知识。
- 项目负责人：在真实路由和响应式壳层内检查信息架构、中文文案与交互，再决定后端契约。

## 3. 信息架构

| 路由 | 页面职责 | 本轮交互 |
| --- | --- | --- |
| `/` | AI 搜索首页 | 多行输入、模式/范围/模型、附件校验、快捷问题、最近搜索、常用空间、数据源概览、上下文面板 |
| `/search` | 搜索结果工作区 | 搜索状态、AI/原始结果切换、引用定位、筛选、收藏、复制、反馈、文档预览 |
| `/research` | 深度研究 | 研究计划、六阶段进度、来源和报告摘要 |
| `/spaces` | 我的知识空间 | 六个空间的权限、负责人、文档与常用搜索概览 |
| `/data-sources` | 数据源 | 八个数据源的连接、同步、权限和管理状态 |
| `/history` | 搜索历史 | 查询、模式筛选、重新搜索、重命名、收藏、删除和批量清理 |
| `/favorites` | 收藏内容 | 类型筛选、标签、备注和删除确认 |
| `/settings` | 搜索设置 | 默认模式、范围、模型和引用偏好本地预览 |

原有 `/knowledge`、`/chat`、`/conversations`、`/downloads` 及全部管理路由继续保留，避免破坏已有能力。

## 4. 核心体验

### 4.1 首页

- 首屏主标题为“今天想查找什么？”，搜索框是唯一主视觉焦点。
- `Enter` 提交，`Shift + Enter` 换行；输入框随内容增高且有清空、附件、模式、范围、模型和发送操作。
- 支持智能搜索、精确检索、深度研究、文档问答四种模式，并显示当前模式中文说明。
- 快捷问题只回填，不在用户未确认时自动提交。
- 右侧上下文桌面可收起；较窄屏幕使用抽屉，不维持三栏布局。

### 4.2 搜索结果

- 查询头部显示问题、模式、范围、数据源数量和“本地模拟，无真实耗时”，禁止伪造精确性能数据。
- AI 答案的固定结构使用语义化 Vue 模板；Markdown 文本必须经过禁用原始 HTML 的解析和 DOMPurify 过滤，禁止把未经清洗的内容传给 `v-html`。
- 关键结论的引用编号可聚焦对应来源，并可打开文档预览抽屉。
- 原始结果支持关键词、类型、数据源、时间、部门、排序和列表密度筛选。
- 文档预览包含元数据、正文、命中高亮、上/下一个命中、复制链接和收藏。

### 4.3 辅助页面

- 深度研究用克制的步骤条表达等待、执行、完成和失败，不使用循环 AI 动效。
- 数据源不得伪装真实连接；每张卡明确显示“模拟数据”及连接/同步/权限状态。
- 搜索历史和收藏删除必须二次确认；所有写操作只改变当前内存状态。

## 5. 状态与边界

核心页面至少覆盖初始、检索中、成功、部分数据源不可用、失败、空结果、权限不足和内容过期。错误文案必须说明下一步动作。

本轮不实现：

- 真实检索、流式生成、上传、下载、导出任务或数据源连接。
- 未经 OpenAPI 确认的 DTO、接口路径、权限码或认证状态；涉及这些契约的界面只展示明确标注的本地模拟数据。
- 服务端 Markdown 清洗；真实 Markdown 接入前必须补充服务端清洗与前端 DOMPurify 二次过滤。
- Token、密钥、密码或真实个人敏感数据的读取、存储和展示。

## 6. 验收标准

- 全部用户可见文案为简体中文，页面和数据中无 Emoji，图标统一来自 `@lucide/vue`。
- 首页主搜索框、四种模式、搜索范围、模型、附件校验、快捷问题和提交链路可操作。
- `/search` 可展示完整 AI 答案、六条引用、十条原始结果、四个相关问题和反馈操作。
- 引用与原始结果能打开/关闭文档预览，抽屉支持 Escape 和焦点返回。
- 新增辅助路由可从桌面和移动完整导航进入；旧路由仍可解析。
- 在 1440px、1280px、1024px、768px、375px 五个视口下，不出现根级横向滚动、关键遮挡或不可达主操作。
- 键盘可完成搜索提交、标签切换和抽屉关闭；交互控件具备可访问名称、清晰焦点环，状态变化通过文字和 `aria-live` 表达。
- Markdown 原始 HTML、脚本、事件属性和危险链接不能进入最终 DOM。
- 真实业务请求为 0，控制台错误为 0。
- `typecheck:web`、`lint:web`、`test:web`、`build:web` 和浏览器验收通过。
