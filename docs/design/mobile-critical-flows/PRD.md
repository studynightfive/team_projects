# 移动端关键流程适配 PRD

## 1. 背景

平台基础页面已经能够在 375px 视口中显示，但现有验收主要覆盖静态页面。用户编辑、知识库上传、答案导出、结果筛选和文档三视图等展开状态仍缺少统一的移动端约束与自动化证据。

## 2. 目标

- 登录、AI 搜索、知识库、文档详情、业务看板、用户管理和上传流程可在 375px 视口完成主要操作。
- 弹窗、抽屉、表格、分页、流式回答与文档三视图不产生页面级横向溢出。
- 手机端主要交互目标高度不低于 44px，并兼容底部安全区。
- 保持 1280px、1440px 桌面布局与既有交互不变。

## 3. 用户故事

### 3.1 普通用户

- 我可以在手机上输入问题、停止流式生成、查看答案与来源，并选择 Markdown、Word 或 PDF 导出。
- 我可以在手机上浏览知识库和文档，并在原文、Markdown、分块三个视图间切换。
- 我可以在个人知识库中点击上传区域选择文件，并调整切分策略。

### 3.2 管理员

- 我可以在手机上查看业务看板和关键指标。
- 我可以打开用户新建或编辑抽屉，完整填写表单并关闭或保存。
- 我可以横向浏览管理表格，同时始终找到当前行的主要操作。

## 4. 范围

- 共享移动端样式：Ant Design Drawer、Modal、确认框、表格滚动区、触控目标和安全区。
- 用户端：AI 搜索、结果筛选、答案导出、知识库目录、上传配置、文档三视图。
- 管理端：业务看板、用户管理及关键管理表格。
- 浏览器验收：375px、1280px、1440px 的静态页面和关键展开状态。

## 5. 非目标

- 不改变后端接口、权限模型、RAG 流程或上传处理逻辑。
- 不重做桌面端视觉设计。
- 不将所有数据表改写为独立卡片组件。
- 不增加新的移动端框架或运行时依赖。

## 6. 验收标准

1. 375px 下根节点 `scrollWidth` 不大于 `clientWidth`。
2. Drawer 和 Modal 的可见边界不超过视口，关闭和主要操作高度不低于 44px。
3. 关键表格保留容器内横向滚动，操作列固定在右侧且可直接操作。
4. AI 搜索输入、停止生成、结果筛选、答案导出可在手机上完成。
5. 上传区和切分配置为单列布局，上传区可点击且触控高度不低于 44px。
6. 原文、Markdown、分块三视图切换按钮均可见且不重叠。
7. 375px、1280px、1440px 浏览器检查无控制台错误。

## 7. 风险

- Ant Design 浮层挂载在 `body`，作用域样式无法覆盖，必须使用受控的全局响应式规则。
- 固定操作列会覆盖表格最后一列背景，需同时设置背景与边缘阴影。
- 浏览器原生文件选择器无法由自动化安全地完成真实文件选择，本次验证上传入口、配置布局和文件输入关联，不上传业务文件。
