# 账号与支持页面 PRD

## 1. 背景与目标

用户工作区顶栏的“通知”“帮助”以及账号菜单中的“偏好设置”目前只显示后续功能提示，管理中心通知入口也没有可进入的页面。本轮将这些入口升级为可直接浏览和操作的本地前端基线，继续沿用 M01 V2 的统一壳层、设计 tokens、响应式和无障碍规则。

本轮目标：

- 用户可查看、筛选并在当前会话内处理通知。
- 管理员可在管理中心查看治理通知，且与用户端复用同一通知页面实现。
- 用户可搜索常见问题并从帮助中心进入已有核心功能。
- 用户可编辑、保存和恢复账号偏好的本地预览，并可切换全局界面主题。
- 所有入口不再显示“后续里程碑开放”的占位提示。

## 2. 用户与场景

- 普通用户：从用户顶栏进入通知中心或帮助中心，从账号菜单进入偏好设置。
- 管理员：从管理顶栏进入管理通知中心，同时保持管理中心壳层。
- 桌面端用户：鼠标移入或键盘聚焦通知铃铛时，先查看最近通知的简略预览。
- 移动端用户：通知和帮助入口保留在顶栏；偏好设置可从个人资料弹窗进入。

## 3. 路由与范围

| 路由 | 页面 | 当前交付 |
|---|---|---|
| `/notifications` | 用户通知中心 | 本地通知列表、搜索、状态/类型筛选、单条与全部标记已读 |
| `/admin/notifications` | 管理通知中心 | 复用通知页面，展示独立的管理通知与未读计数 |
| `/help` | 帮助中心 | 常用入口、分类筛选、关键词搜索和原生折叠问答 |
| `/preferences` | 偏好设置 | 全局界面主题切换，以及工作台、内容呈现和通知偏好的本地编辑、保存与恢复默认值 |

## 4. 核心体验要求

### 4.1 通知中心

- 桌面端通知铃铛在鼠标移入或键盘聚焦时展示最近 4 条通知；预览卡片的水平中心与铃铛中心对齐，并包含标题、类型、时间、未读状态、“全部已读”操作和“查看全部通知”入口。
- 预览中的“全部已读”同步清除当前区域的未读状态并更新铃铛；没有未读通知时按钮禁用。
- 点击铃铛仍进入完整通知中心；点击预览通知进入其已有内部目标并同步标为已读。
- 767px 以下不展示悬停预览，继续通过铃铛直接进入通知中心。
- 顶栏未读数量与通知中心的当前会话状态同步；未读清零后隐藏红点。
- 支持全部、未读、已读三种状态筛选，以及通知类型和关键词筛选。
- 单条“标为已读”和“全部标为已读”只修改内存状态，不调用接口、不写浏览器存储。
- 有业务目标的通知提供真实内部路由；没有目标的通知不渲染无效链接。
- 筛选无结果时显示可访问空状态。

### 4.2 帮助中心

- 提供 AI 搜索、知识库和账号安全三个常用入口。
- 支持按问题标题、答案和分类搜索。
- FAQ 使用原生 `details/summary`，键盘可直接展开和收起。
- 不提交工单、不打开外部客服、不声称连接真实支持系统。

### 4.3 偏好设置

- 提供“跟随系统”“浅色”“深色”三种界面主题；选择后立即应用到用户端、管理端、登录页和错误页。
- 主题选择仅在当前浏览器保存一个非敏感枚举值；刷新或切换路由后保持，选择“跟随系统”时响应系统配色变化。
- 提供默认工作入口、内容密度、链接打开方式、通知摘要和业务通知开关。
- 安全通知保持启用且不可关闭，避免界面暗示可禁用关键安全提醒。
- 业务偏好的保存仍只影响当前页面会话；刷新后恢复固定样例。恢复默认同时将全局主题恢复为“跟随系统”。
- 页面明确展示未保存状态和当前偏好摘要。

## 5. 非目标与接口边界

- 不创建通知、帮助或偏好 API 路径、DTO、权限码或 OpenAPI 字段。
- 不把通知、业务偏好或任何敏感数据写入 `localStorage`、`sessionStorage`、URL 或日志；仅允许主题模式写入固定的 `localStorage` 键。
- 不实现邮件、浏览器推送、工单、客服会话、自定义背景图片、自定义颜色或真实首页重定向。
- 不把本地操作标记为服务端保存成功。
- 管理端“系统设置”和真实退出仍不在本轮范围内。

## 6. 验收标准

- 四个正式路由可直接打开；顶栏、账号菜单和移动端入口指向正确路由。
- 用户端初始 3 条未读、管理端初始 6 条未读；标记已读后同一壳层顶栏计数同步。
- 用户端和管理端的通知铃铛在 1440px、1280px 下可悬停显示最近 4 条通知，预览与铃铛水平居中且不越过视口；键盘聚焦同样可见，“全部已读”与顶栏和完整通知页状态同步。
- 375px 下不显示悬停预览，铃铛仍可直接进入对应通知中心。
- 通知筛选、帮助搜索、FAQ 展开、偏好编辑/保存/重置均可用。
- 跟随系统、浅色和深色模式可切换；主题在用户端、管理端、登录页和错误页全局一致，刷新后仍保持用户选择。
- 每页只有一个 `h1`，顶栏标题与路由一致，无错误侧栏高亮。
- 1440px、1280px 和 375px 下无关键遮挡、重叠和根级横向溢出。
- 浏览器控制台错误为 0，业务 `/api` 请求为 0；未主动切换主题时浏览器存储写入为 0，切换后仅存在一个合法主题键。
- typecheck、Lint、测试、生产构建和浏览器验收通过。
