# 管理与检索体验修正说明

## 目标

本轮在不改变现有核心 API 路径的前提下，完成用户角色单选、搜索入口精简、真实检索评测、默认重排和文档预览/上传体验修正。

## 验收标准

1. 用户管理的新建与编辑只允许选择一个非超级管理员角色；知识库编辑者继承普通用户权限。
2. 管理中心不再展示快捷入口。
3. 测试集编辑器不再提供固定医疗问题填充；每次运行都真实调用检索，不复用历史结果。
4. “检索候选”明确用于查找并标注相关片段，标注结果作为命中率计算的标准答案。
5. 管理端文档列表可进入真实文档预览页。
6. AI 搜索仅保留统一智能搜索，不展示模式切换或附件检索。
7. 请求启用重排但未指定模型时，后端自动选择已启用的重排模型，并在生成答案前完成重排。
8. 知识库详情页为有上传权限的用户提供可点击、可拖拽的上传区域。

## 约束与风险

- 保留 `role_ids` 数组字段以兼容已发布 OpenAPI，但请求只允许一个角色。
- 未配置可用重排模型时，检索仍可完成，并保留未重排顺序；管理端应明确配置至少一个启用的重排模型。
- 命中率依赖测试集中的相关片段标注质量；未标注正确片段时，指标不能代表真实业务效果。

## 验证

- 前端类型检查、ESLint、Vitest、生产构建。
- 后端 Ruff、Mypy、用户管理与 RAG 相关测试、全量 Pytest。
- 本地 Docker 重建后检查登录、用户角色、文档上传/预览、AI 搜索与命中率测试链路。
