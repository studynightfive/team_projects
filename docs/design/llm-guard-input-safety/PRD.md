# LLM Guard 输入安全 PRD

> 状态：已于 2026-07-22 撤销。模型扫描对正常医疗问题存在误判，当前方案见
> [撤除后端 LLM Guard 模型扫描 PRD](../remove-llm-guard-backend/PRD.md)。本文仅保留为历史记录。

## 背景

当前 RAG 输入守卫是本地关键词匹配，只能拦截少量明确词语，不能识别语义化违规主题或
提示词注入。本次接入 Protect AI `llm-guard`，并按项目范围只启用 `BanTopics` 与
`PromptInjection`。

## 目标

- 拦截涉黄、涉赌、涉毒主题及提示词注入。
- 覆盖检索、问答、流式问答、会话首问和用户追加消息入口。
- 在答案缓存查询、向量检索、数据库写入和外部模型调用之前完成检查。
- 模型只在每个 FastAPI 进程中加载一次，推理不阻塞异步事件循环。
- 日志只记录扫描器、类别、分数和错误类型，不记录用户原始输入。

## 验收标准

1. 关键词明显命中时由本地规则快速返回统一 `422`。
2. `BanTopics` 或 `PromptInjection` 判定无效时返回统一 `422`，不访问 RAG 下游。
3. 扫描器加载或推理失败时默认返回 `503`，不静默放行。
4. 多个请求复用同一组扫描器实例，推理在专用线程池执行。
5. FastAPI 启动后后台预热模型，不阻塞健康检查；首次请求会等待同一单例加载完成。
6. `TESTING=true` 时不下载真实模型，单元测试通过依赖替换覆盖模型行为。
7. 只加载 `BanTopics` 与 `PromptInjection`，不加载 Toxicity 或输出扫描器。

## 边界与风险

- 本功能是输入侧风险控制，不替代账号权限、审计、人工复核和模型输出治理。
- 主题模型可能误判医疗语境，阈值必须通过项目真实问题集校准。
- 首次下载模型权重需要网络和较大磁盘空间；Docker 使用独立持久化模型缓存卷。
- 单容器保持一个 Uvicorn Worker，通过容器副本横向扩展，避免同容器重复加载权重。
