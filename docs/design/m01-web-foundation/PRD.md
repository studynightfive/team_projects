# M01 Web Foundation PRD

## 1. 文档状态

- 对应里程碑：M01 `web-foundation`
- 对应设计门禁：Issue #9
- 状态：已由项目负责人批准并通过 PR #10 合入 `main`，作为 M01 实施基线生效
- 产品范围：统一登录入口、普通用户工作区壳层、管理中心壳层、通用状态

## 2. 要解决的问题

统一前端后，普通用户与管理员共用一个 Vue 应用。M01 需要先固定布局、状态、响应式和视觉变量，使后续 14 个里程碑不会分别创造导航、状态页或移动端规则。

这一步影响真实用户的可理解性和可靠感：用户应始终知道自己位于普通工作区还是管理中心，页面正在加载、为空、失败、无权限或不存在时也要有明确下一步。

## 3. 用户与价值

### 3.1 未登录用户

看到唯一登录入口和清晰产品名称，不会在“用户端/管理端”之间做无意义选择。

### 3.2 普通用户

进入普通工作区壳层；主要导航稳定、当前区域清楚、移动端关键入口可达。

### 3.3 具有管理权限的用户

仍可使用普通工作区，并能进入视觉上明确区分的管理中心；管理中心提供“返回用户工作区”入口。

M01 只展示这些路径的静态结构，不判断真实权限。

## 4. 范围

### 4.1 包含

- 统一登录入口视觉骨架。
- 普通用户工作区布局与导航骨架。
- 管理中心布局与导航骨架。
- 加载、空、通用错误、403、404。
- 1440px、1280px、375px 响应式规则。
- 键盘焦点、对比度、触控尺寸和长文本边界。
- design-only 固定 Mock 与验收截图。

### 4.2 不包含

- 真实登录、退出、刷新、`/me` 或 HttpOnly Cookie 行为。
- 用户/管理员路由守卫、权限码和按钮权限。
- 真实知识库、文档、检索、问答、会话、导出或管理数据。
- API 字段、错误码或后端联调完成状态。
- 品牌 Logo、插画、营销页面、复杂图表和动画。

## 5. 信息架构

### 5.1 统一登录入口

- 文字产品名：智能知识库平台。
- 标题：登录工作区。
- 字段标签：账号、密码。
- 主操作：进入工作区。
- 辅助说明：使用组织账号登录；M01 仅固定视觉，不处理凭据。

### 5.2 普通用户工作区

已知 P0 导航顺序：

1. 工作台。
2. 知识库。
3. 检索。
4. 智能问答。
5. 历史会话。
6. 我的下载。

内容区展示静态总览、知识库样例和最近文档，只验证栅格、表格和长文件名表现。它们不是业务功能完成证明。

### 5.3 管理中心

已知 P0 导航顺序：

1. 系统概览。
2. 用户与角色。
3. 模型管理。
4. 知识库管理。
5. 文档与任务。
6. 命中率测试。
7. 审计日志。

顶栏明确显示“管理中心”，并提供“返回用户工作区”。静态总览只验证卡片、服务状态与列表密度。

## 6. 系统状态与文案

| 状态 | 标题 | 说明 | 主操作 |
|---|---|---|---|
| 加载 | 正在加载页面 | 请稍候，正在准备工作区内容。 | 无 |
| 空 | 暂无内容 | 当前范围内还没有可显示的内容。 | 刷新页面 |
| 错误 | 暂时无法加载 | 网络或服务暂时不可用，请稍后重试。 | 重新加载 |
| 403 | 你没有访问权限 | 当前账号不能查看此页面，数据不会被加载。 | 返回工作区 |
| 404 | 页面不存在 | 地址可能已变化，或页面已被移除。 | 返回工作区 |

失败状态不得显示内部堆栈、存储路径、请求体或凭据。403 不显示伪成功内容，也不暗示只靠前端隐藏完成鉴权。

## 7. 响应式要求

### 7.1 1440px

- 240px 固定侧栏、64px 顶栏。
- 内容区最多四列摘要卡片。
- 主表格不溢出页面；长文件名截断并保留完整可访问名称的实现空间。

### 7.2 1280px

- 保持桌面侧栏和顶栏。
- 摘要卡片允许收敛为三列或两列。
- 顶栏操作不得挤压页面标题。

### 7.3 375px

- 侧栏完全隐藏，不以覆盖层挡住首屏主要内容。
- 顶栏显示菜单按钮、当前区域和用户入口。
- 菜单按钮展开完整模块导航；底部导航只保留高频入口。
- 普通工作区提供底部移动导航；管理中心提供紧凑模块入口。
- 摘要卡片、列表和状态卡单列排列。
- 页面宽度不得产生横向滚动；主要操作最小触控高度 44px。

## 8. 可访问性

- 正文与背景对比度目标不低于 4.5:1；大号文字不低于 3:1。
- 所有交互元素保留清晰 `:focus-visible` 焦点环。
- 不只依赖颜色表达成功、警告或错误，同时显示文字。
- 页面使用一个主标题；导航、主要内容和状态区域具备语义标签。
- 动画不是理解状态的必要条件，并尊重减少动态偏好。

## 9. 固定数据与隐私

- 所有名称为明确的演示内容，不是真实组织或个人数据。
- Mock 不包含邮箱、手机号、账号 ID、权限码、Token、Cookie、密钥或下载地址。
- 字段不映射任何未确认 OpenAPI；M02 以后由员工 3 的契约替换。
- 长文本和长文件名只用于布局边界测试。

## 10. 验收标准

- 静态 artifact 可离线打开并切换 user、admin、states 三种视图。
- 七张验收图来自同一提交的可编辑视觉源，像素尺寸符合清单。
- 三种尺寸无文字截断导致的信息丢失、组件重叠、关键操作遮挡或页面横向溢出。
- 登录、加载、空、错误、403、404 均有明确下一步或明确无操作原因。
- 普通与管理布局可一眼区分；管理中心具备返回用户工作区入口。
- tokens、Mock、artifact、截图和文档互相一致。
- 项目负责人批准产品文案、视觉值和无自定义资产决定；员工 1 确认可实现；员工 6 确认运行与证据可重复。

## 11. 失败条件

以下任一情况发生即不能作为 M01 DoR：

- 用 PNG 替代可编辑视觉源。
- 把 Mock 或候选路径称为真实 API Contract。
- 375px 依赖横向滚动或侧栏遮挡主要操作。
- 使用未记录来源的 Logo、图片、字体或远程资源。
- 视觉值尚未获非作者批准却标记为正式基线。
