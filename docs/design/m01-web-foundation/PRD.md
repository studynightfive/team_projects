- 实施取舍：[`V2-Implementation-Spec.md`](V2-Implementation-Spec.md)。
- 状态：项目负责人已于 2026-07-16 明确批准并冻结为 M01 V2 正式视觉与交互基线，供同一长期功能分支的后续里程碑继承。

## 2. 要解决的问题

V1 只证明统一 Vue 工程、两类壳层和安全边界可运行，页面仍带有开发模板感，菜单和操作缺少清晰反馈。V2 要把三张核心页面提升为克制、专业、强信息层级的企业级 SaaS 界面，并让后续子功能继承同一套布局、色彩、组件和响应式规则。

## 3. 用户与价值

### 3.1 未登录用户

通过 55:45 分屏页面理解产品价值和安全定位，并在不暴露凭据的前提下完成账号、密码、密码显隐、记住我和登录方式的界面交互。

### 3.2 普通用户

在一个页面内看到动态问候、知识指标、待办、最近知识库和团队动态；顶栏搜索与侧栏层级清楚，桌面侧栏可折叠，移动端入口可达。

### 3.3 管理员

在管理中心快速理解活跃用户、内容、任务和治理风险，并查看服务健康、治理事项与审计日志；紫色只用于管理身份和关键动作，不形成大面积装饰。

## 4. 本轮范围

### 4.1 登录页

- 桌面为深色品牌区 55% + 白色表单区 45%。
- 品牌区包含平台标记、产品名、主标语、三项价值、版权和界面模板版本。
- 表单包含账号、密码、密码显隐、忘记密码、记住我、企业 SSO、主登录、三种替代入口和法律声明。
- 空提交显示中文错误并把焦点移到第一个无效字段。
- 所有动作仅提供本地反馈，不提交、记录或持久化账号和密码。

### 4.2 普通用户工作台

- 240px 侧栏可折叠为 64px；导航使用 Lucide 线性图标。
- 56px 顶栏包含面包屑、全局搜索、`Ctrl/Cmd + K`、通知、帮助、管理入口和头像。
- 主体包含动态时间问候、两个操作、四张指标卡、最近访问知识库和团队动态。
- 尚未建设的子功能必须明确提示后续开放，禁止无反馈的空链接。

### 4.3 管理中心

- 与用户端共享深色侧栏结构，以 `#553C9A` 标识选中态和关键按钮。
- 顶栏包含管理层级、环境选择、通知、头像和返回用户工作区。
- 主体包含时间筛选、刷新、导出、四张带七点 Sparkline 指标卡、三项服务健康、三项治理事项和紧凑审计表。
- 分页包含可见禁用态；审计表使用文档保留 IP 地址，不展示真实内部地址。

### 4.4 通用状态

- 加载、空、错误、403 和 404 保留安全中文文案和明确下一步。
- 状态图标统一使用 Lucide，不再使用汉字、数字或标点模拟图标。

## 5. 明确非范围

- 真实登录、退出、找回密码、SSO、第三方认证或会话刷新。
- 用户/管理员路由守卫、权限码、真实个人资料或后端鉴权替代。
- 知识库、检索、问答、文档、导出、治理和审计业务 API。
- 正式企业 Logo、官方企业微信/钉钉/飞书品牌资产和真实协议 URL。
- 未确认 OpenAPI、虚构 DTO、下载地址或网络成功态。

## 6. 设计语言

- 用户端主色 `#2B6CB0`，管理端主色 `#553C9A`。
- 页面底色 `#F8FAFC`、卡片 `#FFFFFF`、表头和行 hover `#F1F5F9`。
- 正文使用 Slate 中性色；不使用纯黑、霓虹、渐变和彩色阴影。
- 卡片圆角 12px、按钮/输入框 8px；阴影只允许 sm/md/lg 三档。
- 单页字号不超过五档语义层级；数字使用 `tabular-nums`。
- 所有普通图标来自 `@lucide/vue`；禁止 emoji。

## 7. 响应式

### 7.1 1920px、1440px、1280px

- 登录页保持 55:45，表单最大宽 360px，页面首屏不产生页面级滚动。
- 工作区侧栏 240/64px，顶栏 56px，主体有效宽度不超过 1280px。
- 四张指标卡保持四列；用户内容区为 6:4，管理运营区为双列。
- 审计表只允许卡片内部滚动，不能推动根节点横向溢出。

### 7.2 375px

- 登录页降级为紧凑品牌头 + 单列表单，允许必要的纵向滚动。
- 工作区侧栏完全隐藏；顶栏显示菜单、区域标题和必要入口。
- 指标、知识库、团队动态、服务和治理内容改为单列。
- 底部保留四项高频导航；主要触控区域不小于 44px。
- 页面根节点无横向滚动，移动抽屉支持遮罩、关闭、Escape、焦点循环和焦点返回。

## 8. 可访问性和安全

- 页面只有一个 `h1`，使用语义化 `header/nav/main/section/table`。
- 图标按钮具备中文可访问名称；通知数量不能只依赖红点表达。
- 输入错误使用 `aria-invalid` 和 `aria-describedby`；密码显隐维护 `aria-pressed`。
- 侧栏折叠维护 `aria-expanded`，折叠后图标仍有 `title` 和可访问名称。
- 不把 Token、账号、密码、Cookie 或用户输入写入 localStorage、sessionStorage、URL、日志、Mock 或截图。
- Mock Adapter 默认拒绝未注册请求；V2 浏览器验收业务 `/api` 请求必须为 0。

## 9. 验收标准

- 三页与本 PRD、V2 tokens 和 design-only Mock 一致。
- 用户与管理页面一眼可区分，但共享布局和组件节奏一致。
- 用户四张指标完整；管理四张卡均有 Sparkline，共 28 个数据点。
- 登录表单默认、hover、focus、error 和 disabled 样式存在。
- 1920、1440、1280、375 基线截图无重叠、根级溢出或关键操作不可达。
- 类型检查、Lint、23 项以上自动化测试、生产构建、依赖审计和浏览器验收通过。
- 控制台错误为 0，业务 API 请求为 0。
- 项目负责人已明确确认本版页面，M01 V2 作为正式里程碑基线供 M02–M14 本地页面继承；合入 `main` 仍是统一前端 P0 的最终交付条件。
