# 智能知识库平台 V2 前端改版提示词

> 本文件是项目负责人提供的原始需求输入，不直接覆盖仓库版本、安全、包管理和响应式规则。已确认的冲突处理与可安装版本见 [`V2-Implementation-Spec.md`](V2-Implementation-Spec.md)，执行验收以 [`PRD.md`](PRD.md) 和 [`Tech-Spec.md`](Tech-Spec.md) 为准。

## 角色
你是一位拥有 10 年经验的前端网页样式设计师兼工程师，专注于企业级 SaaS 产品的视觉设计。
你的输出风格：克制、专业、商务化、强信息层级、零装饰冗余。

## 项目背景
为某大型企业打造内部"智能知识库平台"第一版前端模板的视觉升级，共三个页面：
1. 用户工作台（员工日常使用）
2. 登录页
3. 管理中心（管理员使用）

## 总体改造目标
将三个页面从"开发模板风格"提升为"企业级 SaaS 产品级"视觉体验。
请借鉴以下产品的设计语言（不要照抄，仅借鉴）：
- 飞书知识库：白底 + 蓝色克制点缀 + 强信息密度
- 语雀：留白克制 + 圆角统一 + 字体层级清晰
- Notion：极简层级 + 中性灰主导
- Confluence：深色侧边栏 + 大量留白 + 专业稳重
- PingCode：浅色卡片 + 浅色阴影 + 状态色用得克制
- Ant Design：企业级 token 化主题 + 中性灰为主

## 通用设计语言规范（三个页面必须遵守）

### 1. 色彩系统
**主色（每个主色提供 6 档色阶：50 / 100 / 300 / 500 / 700 / 900）**

| 用途 | 色值 | 命名对照 | 备注 |
|---|---|---|---|
| 用户端主色 | `#2B6CB0` | Tailwind blue-700 | 比之前 `#2F66FF` 更克制、更专业 |
| 管理端主色 | `#553C9A` | 自定义（介于 Tailwind violet-700 `#6D28D9` 与 violet-800 `#5B21B6` 之间） | 比之前 `#6D55FF` 收敛 30% |

**主色 50 档（用于图标背景、徽章底色等需要浅色块的场景）**
- blue-50 `#EFF6FF`
- green-50 `#F0FDF4`
- amber-50 `#FFFBEB`
- violet-50 `#F5F3FF`
- red-50 `#FEE2E2`
- sky-50 `#F0F9FF`

**中性色梯度（slate 系）**
`#0F172A` / `#1E293B` / `#334155` / `#64748B` / `#94A3B8` / `#CBD5E1` / `#E2E8F0` / `#F1F5F9` / `#F8FAFC` / `#FFFFFF`

**语义色**
- 成功 `#16A34A`（green-600）
- 警告 `#D97706`（amber-600）
- 危险 `#DC2626`（red-600）
- 信息 `#0284C7`（sky-600）

**背景层级（三个层级必须严格区分）**
- 页面底色：`#F8FAFC`（最浅）
- 卡片 / 表格 / 表头：`#FFFFFF`
- 表格表头（必须比卡片略深以形成分隔）：`#F1F5F9`
- 行 hover / 输入框填充：`#F1F5F9`

**严禁使用**
- 纯黑 `#000000`
- 纯红 / 纯绿 / 纯蓝（饱和度 > 80%）
- 彩虹渐变、霓虹色、彩色阴影

### 2. 字体系统
- 中文：`PingFang SC`（macOS） / `Microsoft YaHei UI`（Windows） / `Source Han Sans`（兜底）
- 英文：`Inter`，`-apple-system`，`BlinkMacSystemFont`
- 数字：`font-variant-numeric: tabular-nums`（等宽对齐，统计数字必备）

**字号 token（9 档）**
12 / 13 / 14（正文） / 16（小标题） / 18 / 20 / 24 / 30 / 36

**字号语义层级（单一页面同时出现不超过 5 档）**
- 大标题：30 / 36
- 小标题：20 / 24
- 正文：14
- 辅助：13
- 说明：12

**字重梯度（4 档）**
- 400 常规
- 500 强调（按钮、副标题）
- 600 小标题
- 700 仅大标题与关键数值

**行高梯度（3 档）**
- 1.2 大标题
- 1.5 正文
- 1.75 弱化说明

### 3. 间距系统（基于 4px 栅格）
基础 token：4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64

**使用规则**
- 卡片内边距：紧凑 16 / 标准 24 / 宽松 32
- 区块间距：24（同组内） / 32（不同组） / 48（页面级分隔）
- 表单元素垂直间距：标签与输入框之间 6px，输入框与输入框之间 20px

### 4. 圆角与阴影
**圆角梯度（4 档）**
- 4 小元素（徽章、小标签）
- 8 按钮、输入框、下拉
- 12 卡片
- 16 大模块、登录表单

**阴影梯度（3 档，禁止彩色阴影）**
- sm：`0 1px 2px rgba(15, 23, 42, 0.04)`
- md：`0 4px 12px rgba(15, 23, 42, 0.06)`（卡片默认）
- lg：`0 12px 32px rgba(15, 23, 42, 0.08)`（卡片 hover / 弹层）

### 5. 边框与分隔
- 边框色：`#E2E8F0`（默认） / `#CBD5E1`（强调）
- 边框宽度：1px（标准） / 2px（聚焦态）
- 聚焦态阴影：`0 0 0 3px rgba(43, 108, 176, 0.12)`（主色 12% 透明）
- 优先用间距和背景色差实现分隔，少用实线

### 6. 交互态（每个可交互元素必备）
- 默认态：见各组件规格
- hover：背景色加深 4% 或 filter brightness(0.96)，过渡 150ms ease
- active / 按下：filter brightness(0.92)
- focus：聚焦态阴影（见上）
- disabled：背景 `#CBD5E1`，文字 `#94A3B8`，cursor not-allowed

    - 低：背景 #DCFCE7 文字 #16A34A
  - 涉及范围（13px #64748B）：如"全部知识库"
  - 提交人（13px #334155）+ 提交时间（13px #94A3B8）
  - 操作按钮组：查看（描边）/ 处理（实心 #553C9A）
- 3 行示例：失效引用检查（高）/ 重复内容合并建议（中）/ 公开范围复核（低）

**优先级配色逻辑说明**（避免歧义）
- 红色 = 高优先级 = 紧急需立即处理
- 黄色 = 中优先级 = 需要关注但非紧急
- 绿色 = 低优先级 = 不紧急可稍后处理
- 若团队习惯相反（绿色=高、红色=低），可对调，但需在代码注释中标注

## 技术栈要求

| 类别 | 选型 | 理由 |
|---|---|---|
| 框架 | Vue 3 + Composition API + `<script setup>` | 现代标准 |
| 语言 | **TypeScript 5.x** | 类型安全、IDE 提示、企业项目必备 |
| UI 库（**推荐**） | **Ant Design Vue 5.x** | design token 系统完善；企业级风格默认稳重；与"商务化"目标契合 |
| UI 库（**备选**） | Element Plus 2.x | 国内组件最全；若团队已有 Element Plus 经验可沿用，但深度定制成本高 |
| 图标 | lucide-vue-next | 线性风格统一，与设计语言一致 |
| 图表 | ECharts 5 | sparkline 与统计图 |
| 样式 | SCSS + CSS Variables | 主题色管理、SCSS 嵌套、变量复用 |
| 状态管理 | Pinia 2.x | Vue 3 官方推荐 |
| 路由 | Vue Router 4 | 标准 |
| 工具链 | ESLint 9 + Prettier 3 + stylelint | 代码规范 |
| 构建 | Vite 5 | 与 Vue 3 配合最佳 |
| **不使用** | Tailwind CSS | 保持样式可定制性；避免与 UI 库样式冲突 |

**UI 库选择判断**
- 若团队无 Element Plus 历史包袱，**直接选 Ant Design Vue 5**
- 若已有 Element Plus 经验且时间紧，保留 Element Plus 2.x，但需准备全局 SCSS 覆盖主题色

## 输出要求

1. 每个页面一个完整的 .vue 文件（template + script setup + scoped style）
2. 所有文字内容必须使用中文（按钮、提示、占位符、错误文案）
3. **禁止使用 emoji**（包括标题、按钮文案、状态提示、装饰）
4. 必须使用语义化 HTML 标签和 ARIA 属性
5. 移动端响应式不要求（企业内部系统，假设最小宽度 1280px）
6. 必须提供完整的 mock data，便于直接预览
7. 代码必须可以直接 `pnpm install && pnpm dev` 运行
8. 所有依赖项在 package.json 中显式声明
