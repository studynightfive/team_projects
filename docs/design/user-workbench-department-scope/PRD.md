# 普通用户 AI 工作台与部门知识库范围修复 PRD

## 背景

普通用户登录后，AI 搜索工作台会调用仅管理员可访问的模型管理接口，接口返回
403 后导致整个工作台加载失败。部分演示账号的部门属性与个人知识库所属部门不一致，
造成用户看到其他部门知识库，却看不到预期部门知识库。

## 目标

1. 已登录普通用户可以正常打开 AI 搜索工作台。
2. 普通用户只能看到自己的个人知识库和本人所在部门的企业知识库。
3. 管理员调整用户部门时，同步更新该用户个人知识库所属部门。
4. 模型管理密钥、参数和管理权限继续仅对管理员开放。

## 验收标准

- 普通用户请求可用聊天模型列表返回 200，不包含密钥状态和管理参数。
- 可用模型接口只返回已启用模型。
- 模型列表暂时不可用时，工作台仍可加载，并使用环境默认模型选项。
- 知识库列表失败与模型列表失败互不连带。
- 医疗部门普通用户只能看到医疗部门企业知识库和自己的个人知识库。
- 修改用户部门后，用户会话失效，重新登录后知识库范围立即按新部门生效。

## 约束与风险

- 不放宽现有 `/api/v1/models` 管理接口权限。
- 不修改知识库服务现有部门过滤规则。
- 演示库中的历史错位数据需要通过管理 API 重新保存部门后修复。
- 部门变更会撤销用户现有会话，用户需要重新登录。
