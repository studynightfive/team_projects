HNSW 算法图解

Hierarchical Navigable Small World：多层邻近图近似最近邻索引。上层稀疏做“高速公路”粗定位， 下层稠密做精细搜索。本仓库 Milvus 后端默认使用 HNSW + COSINE（参数 M / efConstruction / ef）。

1. 多层图结构（Hierarchy）

Layer 2（稀疏） Layer 1 Layer 0（全量） 查询入口 / 当前节点 候选近邻

2. 搜索流程（贪心 + 候选集）

3. 插入流程（随机层高）

与本仓库的关系
默认向量后端仍是 PostgreSQL + pgvector。设置 VECTOR_BACKEND=milvus|dual 时， Milvus collection 使用 HNSW + COSINE，参数见 MILVUS_HNSW_*。
调参直觉
召回不足 → 增大 ef / efConstruction；内存或建库慢 → 减小 M； 延迟过高 → 减小 ef，或先用 dual 旁路验证再切 milvus。
为何多层
单层 NSW 容易陷入局部最优。分层后上层“跳得远”，下层“搜得细”， 复杂度近似对数级，适合大规模向量检索。
