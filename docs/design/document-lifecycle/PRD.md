# 文档批量生命周期管理 PRD

## 1. 背景

当前知识库已经支持真实文档上传、异步解析、Markdown 标准化、切分和向量化，但前端上传文件后立即提交并关闭操作区，文档只能逐条删除、逐条重新处理，删除后也无法恢复。Embedding 模型切换后，管理员缺少明确的批量重建索引入口和可核验进度。

“我的空间”仍混有成员、热门问题和设置等未接入真实接口的栏目，双栏详情布局在中等宽度下容易挤压知识库卡片，真实空间信息也使用了固定负责人和部门占位。

## 2. 目标

1. “我的空间”只展示真实可访问知识库，以稳定的列表布局提供搜索、分类、文档规模和直达操作。
2. 上传文件先进入本地队列，用户确认后统一提交；单次至少支持 5 个文件，受后端最大文件数和总大小约束。
3. 管理员可勾选多个文档批量重新处理，并看到每个文档及整体真实进度。
4. 管理员可一次勾选至少 10 个文档批量删除。
5. 删除进入回收站并立即退出 RAG 检索；默认保留 30 天，期间可一键恢复。
6. 个人知识库所有者可批量上传和重新处理自己的文档，不能管理其他个人知识库。

## 3. 用户故事

- 作为知识库管理员，我更换 Embedding 模型后，可以勾选知识库内文档统一重建向量，并确认每一项的完成或失败状态。
- 作为知识库管理员，我可以先拖入多份文件、检查队列和切分参数，再一次确认上传。
- 作为知识库管理员，我可以批量删除错误文档，并在误删后从回收站恢复。
- 作为普通用户，我可以在“我的空间”快速找到所在部门和个人知识库，不会看到无真实数据支撑的成员或热门问题。

## 4. 验收标准

### 4.1 我的空间

- 1440px、1280px 和 375px 无页面级横向滚动或内容重叠。
- 真实 API 模式不展示固定成员、固定负责人、热门问题或空间设置。
- 每个空间可直达文档列表或携带该知识库范围发起搜索。

### 4.2 批量上传

- 点击或拖拽文件只加入队列，不立即发送请求，也不关闭抽屉或上传区域。
- 队列支持追加、移除、清空和重复文件提示。
- 5 个以上合法文件可统一提交；提交期间禁止重复操作，完成后展示逐项结果。
- 上传切分参数应用于本批次全部文件。

### 4.3 批量重新处理

- 管理端文档页和具备管理能力的知识库页可勾选文档并发起批量重新处理。
- 后端为每个文档创建真实文档任务；前端轮询任务接口显示逐项和整体进度。
- 处理中、已删除或越权文档不能被重新处理，错误逐项返回且不影响合法项。
- 界面明确提示更换 Embedding 模型或调整切分参数后需要重新处理。

### 4.4 批量删除与回收站

- 可一次勾选至少 10 个文档批量删除。
- 删除后文档从普通列表消失，活跃分块和 RAG 检索投影立即撤下。
- 回收站显示删除时间和预计清理时间，默认 30 天。
- 恢复后文档退出回收站并自动进入重新处理；可查看真实处理进度。
- 所有批量操作沿用个人/部门知识库服务端权限边界。

## 5. 非目标

- 本期不提供超过保留期后的人工恢复。
- 本期不实现跨知识库移动文档。
- 本期不允许为同一批次的不同文件配置不同切分策略。
- 本期不新增独立批处理任务表，批量进度由现有文档任务聚合。

## 6. 风险与约束

- 删除与正在运行的 Worker 可能并发，Worker 必须在写入新索引前再次确认文档未被删除。
- 回收站保留原文件、Markdown 和历史分块，会增加存储占用；超过保留期后由 Worker 周期清理。
- 恢复必须重新处理，避免在 Embedding 模型已经切换时重新启用旧向量。
- 批量接口最多接受 100 个文档 ID，并去重，避免一次请求制造无界任务。
