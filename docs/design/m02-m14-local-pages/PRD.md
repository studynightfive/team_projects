# M02–M14 本地页面 PRD

## 1. 目标

在后端契约尚未提供时，先把统一前端的完整信息架构、页面结构和主要局部交互做成可运行、可点击、可截图验收的本地版本。这样项目负责人可以连续调整样式和内容，后续接入 OpenAPI 时无需重新设计页面。

本轮成功不等于业务联调成功。核心价值是把导航、内容层级、响应式和交互反馈提前稳定下来，同时保留安全的接口边界。

## 2. 用户与场景

- 普通用户：浏览知识库和文档、检索、问答、维护历史会话、查看导出任务。
- 管理员：在同一应用进入管理中心，查看概览并管理用户、角色、模型、知识库、文档、任务、检索测试和审计日志。
- 项目负责人：直接在本地浏览全部页面，对登录、用户端和管理端基线做连续调整。

## 4. 核心体验要求

### 4.1 导航

- 正式路由表中的每个页面都能从桌面侧栏或移动完整导航进入。
- 用户端底部导航保留四个高频入口；管理端底部导航保留四个高频入口。
- 动态详情页保持父导航高亮；“用户与角色”“文档与任务”分别覆盖两个路由。
- 顶栏标题和面包屑随当前路由变化，不能一直显示“工作台”或“平台总览”。

### 4.2 页面反馈

- 每页只有一个 `h1`，标题区包含页面说明和主要操作。
- 写操作按钮明确标记为“本地预览”，只改变当前页展示状态，不伪装服务端成功。
- 表格在卡片内部横向滚动；根节点不得横向溢出。
- 页面至少覆盖主数据状态，并对空、失败、403 或过期等关键状态提供可访问文案。

### 4.3 响应式与无障碍

- 1920、1440、1280 和 375px 下无关键遮挡、重叠或根级横向溢出。
- 767px 以下使用 M01 移动抽屉和底部导航，主要触控区域不小于 44px。
- 图标带可访问名称或隐藏属性；表单有显式标签；键盘可完成导航、筛选、弹窗关闭和主要操作。
- 继续支持 reduced motion，并保留 M01 的焦点环、抽屉焦点返回和滚动锁行为。

## 5. 非目标

- 不新增 `/profile`；个人资料继续通过账号菜单/后续抽屉承载，待正式路由批准后再调整。
- 不新增用户、角色、模型、知识库等详情路由；本轮使用页面内面板、弹窗或抽屉。
- 不创建 API DTO、权限码实现、认证 Store、路由守卫、SSE 解析器、上传器、下载地址或轮询器。
- 不把固定 Mock 当作真实业务数据，不在日志、URL、localStorage 或截图中放入 Token、密码或密钥。

## 6. 验收标准

- 正式路由表中的 19 个业务入口和 M01 的登录/403/404 均可直接打开；动态路由使用固定样例 ID。
- 所有桌面和移动导航项可点击，无 `href="#"`、静默按钮或错误父菜单高亮。
- M01 tokens、壳层、按钮、卡片、表格、焦点和响应式规则一致。
- 搜索、筛选、标签页、分页、确认、抽屉和局部状态在刷新前可操作，刷新后恢复固定数据。
- 浏览器控制台错误为 0，业务 `/api` 请求为 0；未读取或写入浏览器凭据存储。
- typecheck、Lint、测试、生产构建和页面级浏览器验收通过。
