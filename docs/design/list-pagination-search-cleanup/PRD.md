# 列表分页与 AI 搜索精简 PRD

## 背景

当前多数业务接口支持分页，但前端常以 `page_size=100` 拉取后完整渲染；数据增长后页面会变慢，也无法明确当前位置。AI 搜索仍保留常用问题和重复引用来源，命中率指标只显示英文缩写，不利于演示理解。

## 目标

1. 命中率测试指标采用“中文名称（英文缩写）”显示。
2. 所有会持续增长的业务数据集合提供分页、总数和每页数量选择。
3. 删除 AI 搜索首页常用问题区域。
4. 引用来源概览按文档去重，并删除“待确认”状态。
5. 完成前后端权限、数据状态、分页边界和异常路径核查。

## 分页范围

用户、角色、部门、模型、知识库、文档、任务、审计日志、通知、收藏、下载、测试集、测试运行、逐题结果、知识空间与原始检索结果。

导航菜单、下拉选项、仪表盘摘要、文档章节目录和上传候选等短暂操作集合不分页。

## 验收标准

- 默认每页 10 条，可切换 10/20/50 条；只有一页时仍显示总数，分页按钮不可越界。
- 搜索或筛选条件变化时回到第一页；删除最后一条后不会停在空白页。
- 引用中同一 `documentId` 只出现一次；缺少文档 ID 时使用标题和来源组合去重。
- 命中率结果不再出现只有英文、没有中文释义的指标标题。
- TypeScript、ESLint、Vitest、生产构建、Ruff、Mypy 和后端测试通过。

## 风险与约束

- 本轮不改变后端指标字段名，避免破坏 OpenAPI；只修改前端展示。
- 需要客户端筛选的列表会循环读取全部后端页，再由统一分页状态展示；接口每次最多读取 100 条，直到达到后端返回的 `total`。
- 仓库规则禁止直接推送 `main`，完成后推送当前功能分支并通过 PR 合并。
