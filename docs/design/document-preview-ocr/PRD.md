# 文档三视图与 OCR 展示 PRD

## 背景

文档上传后已经保留原件、生成标准 Markdown 并写入分块，但用户详情页目前只展示
Markdown，无法核对原文、切分结果和 OCR 质量。管理员和知识库用户需要在同一页面完成
“原件核对、转换核对、检索分块核对”。

## 目标

- 文档详情默认展示原文，并可切换原文、Markdown、分块三个视图。
- 原文、Markdown、分块均复用知识库访问权限，不暴露宿主机存储路径。
- 展示 OCR 是否执行、识别语言、平均置信度和人工复核提示。
- 分块列表展示标题、页码、字符范围、Token 估算和向量状态，并支持分页。
- 后台处理继续使用 Markdown 作为切分与索引的标准中间格式。

## 非目标

- 不引入在线 Office 转换服务。
- 不更换现有解析器、Chunker、Embedding 或向量数据库。
- 不把 OCR 结果或原始文件开放为无鉴权静态地址。
- 不承诺浏览器原生预览 DOC、DOCX、XLSX 或 PPTX。

## 用户流程

1. 用户从有权访问的知识库打开文档详情。
2. 页面默认读取原文；PDF、图片和文本直接内联预览。
3. Office 文件显示原件信息和下载操作，不伪造转换后的原版式。
4. 用户切换到 Markdown，查看清洗后的标准化正文。
5. 用户切换到分块，分页检查检索分块及向量状态。
6. OCR 质量需要复核时，页面在三个视图上方持续显示提示。

## 验收标准

- 无权用户请求详情、原文、Markdown、分块均返回 403。
- 原文接口使用鉴权响应，不返回或记录内部存储路径。
- Markdown 经过服务端清洗并由前端 DOMPurify 二次过滤。
- 分块接口默认每页 20 条，页码和页大小参数受限。
- OCR 状态至少区分未启用、无需 OCR、已完成、低置信度和不可用。
- PDF、图片、文本可预览；Office 可下载原件。
- 加载、处理中、空、403、404、网络失败和长文本状态可用。
