# RAG 问答导出与下载记录 PRD

## 背景

AI 搜索页当前把问答结果即时导出为临时 Markdown 文件。文件响应结束后即被删除，
不会写入“我的下载”，用户也无法在导出前选择 PDF、Word 或 Markdown。

## 目标

- 用户在 AI 搜索结果页选择 PDF、Word（DOCX）或 Markdown。
- 支持的浏览器打开系统保存位置选择器；其他浏览器使用标准下载行为。
- 每次真实 API 问答导出都写入当前用户的导出任务记录。
- “我的下载”能够区分问答导出与文档导出，并可再次鉴权下载。
- 导出内容只包含当前问题、生成答案和引用摘要，不打包参考文档正文。

## 验收标准

1. 未生成答案时不能发起导出。
2. 取消保存位置选择时不请求后端、不创建任务记录。
3. PDF、DOCX、Markdown 均能生成非空文件，响应文件名与格式一致。
4. 导出成功后，`GET /api/v1/exports` 返回对应的已完成问答导出记录。
5. 记录的下载地址只能由任务所属用户使用，且过期、签名和目录边界检查保持有效。
6. “我的下载”展示真实文件名、格式、状态和创建时间，并可再次下载。
7. 导出失败保留失败记录，接口返回统一错误，不泄露服务端路径或异常堆栈。

## 边界

- 浏览器禁止网页静默指定任意本地目录。支持 File System Access API 时使用系统保存选择器，
  不支持时回退到浏览器标准下载流程。
- 本次不新增数据库表或迁移，复用 `export_tasks`；`document_ids=[]` 表示问答结果导出。
- 不改变已有文档批量导出和 Worker 执行逻辑。
