# 产品上线发布检查清单

本文档规定软件产品从预发布到正式上线的最低检查项，适用于 Web 与移动端业务系统。发布负责人须在上线窗口开始前两小时完成清单勾选，并将结果同步至值班群。

发布前：确认变更单已审批；确认目标环境分支与镜像标签；确认数据库迁移已在预发演练成功且可回滚；确认配置中心开关默认安全；确认监控告警、日志采集与链路追踪探针正常；确认回滚方案、回滚负责人与预计耗时已写明。

功能验证：核心下单、支付、登录与权限链路至少各抽检三条成功路径与一条失败路径；对涉及金额与库存的接口核对幂等键；对新增页面完成桌面与移动端基本可用性检查。

安全与合规：无明文密钥写入仓库或镜像；对外接口限流策略已启用；个人敏感信息脱敏规则未回退；第三方回调签名校验保持开启。

上线中：按变更单顺序执行；每个关键步骤完成后在清单记录时间戳；出现错误率超过基线两倍或核心接口可用性低于百分之九十九点五时，立即暂停并启动回滚评估。

上线后二十四小时内：发布负责人持续关注错误日志与业务看板；发现问题先止血再归因；形成《上线复盘》归档到知识库「运维复盘」目录。未完成复盘的发布不得关闭变更单。

本清单由研发效能组维护，重大变更须增补专项检查项并由架构评审确认。
