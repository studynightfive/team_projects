# 深度研究模块移除修正记录

更新时间：2026-07-19

## 背景

当前后端没有独立的深度研究任务接口、任务表或异步编排服务。为了保证明天交付演示聚焦真实可用能力，用户端不再保留“深度研究”页面和入口，避免把固定样例误认为真实接口能力。

## 修正内容

1. 删除用户端深度研究页面：
   - 删除 `/research` 路由。
   - 删除 `frontend/web/src/views/user/DeepResearchView.vue`。

2. 清理可见入口：
   - 从用户侧栏和移动抽屉中移除“深度研究”。
   - 从首页快捷任务中移除“创建深度研究”。
   - 从 AI 搜索模式选项中移除“深度研究”，保留智能搜索、精确检索和文档问答。

3. 清理类型与样例残留：
   - 移除前端 `Research`/`ResearchStatus` 类型。
   - 移除 Mock 中的研究任务对象。
   - 将旧 Mock 搜索历史中的 `research` 模式归并为 `smart`。
   - 更新相关单测断言和导航数量。

## 大修说明

本次没有新增后端接口，也没有改动真实 RAG 搜索链路。删除范围集中在前端深度研究模块及其 Mock 类型，目的是降低交付范围和演示风险。

## 验证记录

- `pnpm.cmd run typecheck:web`：通过。
- `pnpm.cmd run lint:web`：通过。
- `pnpm.cmd run test:web`：通过。
- `pnpm.cmd run build:web`：通过。

## 剩余限制

后续如需恢复“深度研究”，应先补后端任务接口、任务状态、步骤结果、引用快照和报告导出契约，再恢复前端页面；不建议继续以固定样例页面承载该能力。
