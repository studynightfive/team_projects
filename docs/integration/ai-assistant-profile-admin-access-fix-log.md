# AI 助手移除、账号资料真实化与管理入口权限修正记录

日期：2026-07-19

## 背景

交付演示前复查发现，用户端同时存在“AI 搜索”和“AI 助手”两个问答入口。真实 API 模式下二者都调用同一条 RAG 回答链路，保留重复页面会增加演示路径和联调解释成本。同时，用户壳层个人资料仍显示固定演示身份，普通用户也能看到进入管理中心的跳转入口。

## 修正内容

1. 移除重复的 AI 助手页面。
   - 删除用户端 `/chat` 和 `/chat/:conversation_id` 路由。
   - 从桌面侧边栏、移动端导航、偏好设置和帮助说明中移除“AI 助手”入口。
   - 历史会话页继续保留，用于查看真实 RAG 问答记录；新建问答统一回到 AI 搜索首页。

2. 账号资料改为真实登录态。
   - 新增前端会话 Store，保存 `login`、`refresh` 或 `/api/v1/me` 返回的当前用户。
   - 用户端和管理端壳层的姓名、角色和头像首字统一读取当前登录账号。
   - 个人资料弹窗的数据来源说明更新为 `/api/v1/me`，不再标记为固定演示资料。

3. 管理中心入口按真实权限控制。
   - 用户端顶部和移动抽屉只有管理员权限账号才展示“管理中心”入口。
   - 真实 API 模式下，路由守卫进入 `/admin` 前会恢复或读取当前用户，并根据管理员权限判断；普通用户手动访问 `/admin` 会进入 403。
