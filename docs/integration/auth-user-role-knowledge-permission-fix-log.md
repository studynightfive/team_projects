# 登录、退出、用户角色编辑与普通用户知识库权限修正记录

日期：2026-07-19

## 背景

交付演示复查发现四个体验与权限边界问题：登录账号密码错误时提示成会话失效；账号菜单中的退出登录仍是后续功能提示；管理员编辑用户时可以修改账号 ID 和姓名；普通用户知识库列表暴露了创建知识库能力。

## 修正内容

1. 登录错误文案修正。
   - 登录接口返回 401 时，登录页显示“账号或密码错误，请重新尝试”。
   - 其他接口的 401 仍保持“登录状态已失效，请重新登录”，用于会话过期场景。

2. 退出登录功能完善。
   - 新增前端 `logoutCurrentUser` 服务，调用 `POST /api/v1/auth/logout`。
   - 退出后清理内存 Access Token 和 Pinia 当前用户，并跳转 `/login`。
   - 即使后端会话已失效，前端也会清理本地登录态，避免继续停留在业务页。

3. 管理员编辑用户收敛为角色编辑。
   - 新建用户时仍填写账号 ID、姓名、初始密码和角色。
   - 编辑已有用户时账号 ID 与姓名只读展示，保存只提交 `role_ids`。

4. 普通用户知识库创建能力移除。
   - 用户端 `/knowledge` 仅展示当前账号可访问知识库。
   - 创建知识库功能保留在管理中心知识库管理页，不再暴露给普通用户页面。
