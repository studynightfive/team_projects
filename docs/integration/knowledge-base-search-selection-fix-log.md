# AI 搜索知识库选择与重名修正记录

## 背景

AI 搜索页面的搜索范围仍使用前端固定样例，用户无法明确选择当前账号真实可访问的知识库；同时企业知识库允许创建重名，容易导致上传、预览和 RAG 检索时选择对象混淆。

## 修正

- 搜索框新增真实知识库下拉选项。
- 真实 API 模式下，AI 搜索首页和搜索结果页都会读取 `/api/v1/knowledge-bases`，并展示当前账号可访问的知识库。
- 用户选择某个知识库后，前端会把该知识库 ID 写入 RAG 请求的 `kb_id`，后端只在该知识库范围内检索。
- 知识库创建与重命名增加名称查重，忽略首尾空格和大小写差异。
- 新增数据库唯一索引迁移，防止并发创建同名知识库。

## 验收

- 搜索框的知识库选项来自真实后端知识库列表。
- 选择指定知识库后，RAG 请求携带 `workspaceId`，后端转换为 `kb_id`。
- 创建或重命名为已有知识库名称时返回冲突提示，不再允许重名。
