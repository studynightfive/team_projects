# 前端功能使用指南

## 1. 运行模式

- 生产构建和 `dev:web:api` 使用真实 FastAPI 接口。
- `dev:web`、Vitest 或显式 `VITE_USE_REAL_API=false` 才使用 Mock Adapter。
- Mock 只验证接口形状和页面状态，不能作为真实业务结果。
- Access Token 只保存在内存，Refresh Token 使用 HttpOnly Cookie；真实模式下未登录会跳转 `/login`。

## 2. 用户工作区

### 2.1 AI 搜索 `/` 与 `/search`

- 首页从真实接口读取当前账号可访问知识库和已启用聊天模型。
- 用户必须选择一份知识库后提问；不提供跨知识库聚合、附件检索、精确搜索、文档问答或固定常用问题。
- 提交后直接进入 `/search` 并携带一次性路由状态，不把问题写入 URL，也不要求再次输入。
- 后端先执行权限过滤、混合检索和可选重排，再由配置的聊天模型根据命中片段生成答案。
- 答案正文、引用和原始结果可打开文档预览；原始结果支持关键词、来源、部门、时间筛选和分页。
- 右侧引用来源按 `documentId` 去重，同一文档只显示一次，不展示“待确认”状态。
- 当前问题与答案可直接导出为 Markdown、TXT 或 DOCX，不会把全部参考文档正文打包进导出文件。
- 用户输入在检索和模型调用前经过涉黄赌毒内容守卫。

### 2.2 企业知识库 `/knowledge`

- 普通用户只能查看和检索所属部门的企业知识库，不能创建企业库或上传企业文档。
- 部门管理员、知识库编辑者和超级管理员按后端权限创建、维护企业知识库。
- 知识库和文档列表使用真实 API，支持搜索、筛选、10/20/50 条分页和总数显示。
- 文档详情读取服务端 Markdown，并经过服务端清洗和前端 DOMPurify 二次过滤。

### 2.3 我的空间 `/spaces`

- 每个用户首次建立账号时拥有唯一个人知识库。
- 普通用户可在自己的个人知识库上传文档、选择切分策略并发起问答。
- 企业知识空间只展示当前用户所属部门可访问内容。

### 2.4 收藏、下载和通知

- `/favorites` 从真实收藏接口读取、修改备注与标签并删除收藏。
- `/downloads` 展示真实异步导出任务；AI 答案优先在搜索页直接导出。
- `/notifications` 和 `/admin/notifications` 分别读取用户和管理员通知，已读状态写入后端。
- 收藏、下载、通知和帮助主题列表均使用统一分页交互。

### 2.5 账号页面

- `/preferences` 保存界面主题等非敏感本地偏好。
- `/settings` 当前是本地搜索展示偏好，因为后端尚无账号级搜索偏好契约；它不会改变真实 RAG 参数。
- `/help` 为静态帮助内容，不提交工单或替代管理员授权。

## 3. 管理中心 `/admin`

管理路由按后端权限码校验。普通用户不能进入管理中心，也不会预加载管理数据。

### 3.1 用户、部门与角色

- 用户管理支持创建账号、分配单一非超级管理员角色、设置部门和重置密码。
- 超级管理员角色不能通过新建或编辑用户下拉框分配。
- 部门由超级管理员创建并指定唯一部门管理员；部门管理员只能管理本部门资源。
- 角色授权展示按模块分组的权限，知识库编辑者默认包含普通用户全部权限。

### 3.2 模型管理

- 支持 DeepSeek、DashScope、Kimi/Moonshot、智谱、MiniMax、豆包、千帆和其他 OpenAI 兼容供应商。
- 聊天、Embedding 和 Rerank 模型分别配置；密钥只允许写入，页面和接口不回显明文。
- 切换 Embedding 模型或维度后必须重新处理文档，不能混用旧向量。

### 3.3 知识库、文档与任务

- 知识库归属于部门或个人所有者；企业库名称在部门内唯一。
- 上传文档时选择 `fixed`、`semantic`、`recursive` 或 `format` 切分策略，并可设置固定长度与重叠字符。
- 文档先解析、清洗并转为 Markdown，再切块、生成 Embedding 并写入 pgvector。
- 文档管理支持查看真实文档详情；任务中心展示处理、重试和失败状态。

### 3.4 命中率测试

- 可以维护带相关 Chunk 标注的测试集，也可以直接输入单个问题即时测试。
- 每次运行都调用真实检索，不复用历史结果；阈值作用于最终检索或重排分值。
- 结果展示命中率（Hit Rate）、平均倒数排名（MRR）、召回率（Recall@K）、准确率（Precision@K）、NDCG@K 和 MAP@K。
- 逐题结果、候选片段和历史运行记录支持分页。

### 3.5 审计日志

- 审计日志来自真实后端，网络标识和请求标识在前端展示前掩码。
- 管理首页只展示最新审计摘要，完整记录在 `/admin/audit-logs` 分页查看。

## 4. 分页约定

- 所有持续增长的业务列表默认每页 10 条，可切换 10/20/50 条。
- 搜索或筛选变化后回到第一页；删除最后一条时自动校正到有效页。
- 需要客户端筛选的接口会按后端 `total` 循环读取全部服务端页，不会只显示前 100 条。
- 导航、下拉选项、仪表盘摘要、文档章节和上传候选不属于业务记录列表，不分页。

## 5. 已移除入口

- AI 助手、深度研究、历史会话、搜索历史和数据源页面已从路由和导航删除。
- 访问旧地址会进入统一 404 页面，不会请求已删除的业务接口。
